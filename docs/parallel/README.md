# 并行任务执行方案

## 概述

本文档描述了并行任务执行功能的设计方案。**并行模式和串行模式将共存，通过不同的启动命令来选择执行模式，不删除原有代码。**

## 背景

当前实现：
- 计划是 steps 线性数组
- 执行是完全串行的 while 循环
- 步骤间通过 `next_step_hint` 暗示依赖关系

## 目标

1. 将计划改为 DAG（单向无环图）结构
2. 支持并行执行独立任务
3. 处理并发 scope 状态管理
4. **保持串行模式兼容，通过启动命令切换**

## 启动方式

```bash
# 串行执行模式（默认，原有行为）
npm start
# 或
node src/index.js

# 并行执行模式
node src/index.js --parallel
# 或
node src/index.js -p
```

环境变量配置：

```bash
# 并行执行配置
MAX_PARALLEL_TASKS=3      # 最大并行任务数
USE_PARALLEL_EXECUTION=true  # 启用并行模式
CONTINUE_ON_FAILURE=false    # 失败后是否继续
```

## 模式对比

| 特性 | 串行模式 | 并行模式 |
|------|----------|----------|
| 计划结构 | steps 数组 | DAG 图 |
| 执行方式 | while 循环 | p-limit 并发 |
| 依赖处理 | next_step_hint | 显式 depends_on + 隐式推断 |
| Scope 更新 | 直接写入 | 锁机制保护 |
| 适用场景 | 简单任务、调试 | 复杂多步骤任务 |

## 关键设计决策

### 依赖声明方式

支持两种依赖声明：
- **显式依赖**：通过 `depends_on` 字段声明
- **隐式依赖**：通过 scope 变量引用自动推断

### 并发控制

- 使用 `p-limit` 控制最大并发数（默认 3）
- 支持配置最大并行任务数
- 支持动态调整并发数

### 状态管理

- Scope 使用锁机制处理并发更新
- 每个步骤维护独立的状态（pending/ready/running/completed/failed）

## 优势

1. **性能提升**：独立任务可并行执行，减少总执行时间
2. **灵活规划**：支持更复杂的依赖关系
3. **资源优化**：可配置并发数适配不同场景
4. **渐进迁移**：可与现有串行执行模式共存
